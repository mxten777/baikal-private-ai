## 1. 개요

### 1.1 제품 소개

**BAIKAL Private AI**는 기업 내부 문서를 기반으로 AI가 질문에 답변하는 **폐쇄망(Air-Gap) 설치형 플랫폼**입니다.

ChatGPT와 같은 외부 AI 서비스를 사용할 수 없는 보안 환경에서, 사내 문서를 안전하게 활용하여 업무 생산성을 높이는 솔루션입니다.

### 1.2 왜 BAIKAL인가?

| 구분 | 외부 AI 서비스 (ChatGPT 등) | **BAIKAL Private AI** |
|------|----------------------------|----------------------|
| 데이터 위치 | 외부 클라우드 서버 | **사내 서버 (100% 온프레미스)** |
| 인터넷 필요 | 필수 | **불필요 (폐쇄망 운영 가능)** |
| 데이터 유출 위험 | 존재 | **제로 — 외부 통신 없음** |
| 사내 문서 기반 답변 | 불가 | **가능 — RAG 파이프라인** |
| 월 사용료 | 유료 (API 과금) | **무료 — 오픈소스 기반** |
| 커스터마이징 | 제한적 | **전면 커스터마이징 가능** |

### 1.3 핵심 가치

```
🔒 보안        외부 API 호출 제로, 데이터가 서버 밖으로 나가지 않습니다
📄 문서 기반    사내 문서를 업로드하면 AI가 학습하여 정확한 답변을 제공합니다
⚡ 실시간      질문 즉시 AI가 실시간 스트리밍으로 답변합니다
🏢 온프레미스   폐쇄망·DMZ·사내망 — 어디든 설치 가능합니다
💰 비용 절감    오픈소스 기반, 라이선스 비용 없음
```

## 2. 주요 기능

### 2.1 AI 질문응답 (RAG)

**"문서를 업로드하고, 질문하세요. AI가 답변합니다."**

- 사내 문서(PDF, Word, Excel)를 업로드하면 AI가 내용을 자동으로 분석합니다
- 자연어로 질문하면 문서 내용을 기반으로 정확한 답변을 생성합니다
- 답변과 함께 **참고 문서 및 출처**를 명시합니다
- **실시간 스트리밍**: 답변이 한 글자씩 표시되어 대기 시간 없이 확인 가능합니다
- **대화 맥락 유지**: 이전 대화를 기억하여 연속 질문이 가능합니다

```
[사용자]  인사규정에서 연차 휴가 일수는 어떻게 되나요?

[AI]      인사규정 제25조에 의하면, 직원의 연차 유급휴가는 다음과 같이
          부여됩니다:
          
          1. 1년 미만 근무자: 월 1일 (최대 11일)
          2. 1년 이상 근무자: 15일
          3. 3년 이상 근무자: 매 2년마다 1일 추가 (최대 25일)
          
          📄 출처: 인사규정_2025.pdf (p.12)
```

### 2.2 문서 관리

**"업로드 한 번으로 AI가 활용할 수 있는 지식이 됩니다."**

| 기능 | 설명 |
|------|------|
| **다중 형식 지원** | PDF, DOCX(Word), XLSX(Excel) |
| **드래그 앤 드롭** | 파일을 끌어다 놓기만 하면 업로드 |
| **자동 분석** | 텍스트 추출 → 청킹 → 벡터 임베딩 자동 처리 |
| **상태 추적** | 업로드/처리중/완료/실패 실시간 상태 표시 |
| **대용량 지원** | 최대 100MB 파일 업로드 가능 |
| **다운로드** | 원본 파일 다운로드 가능 |

### 2.3 문서 검색

**"원하는 정보를, 원하는 방식으로 찾으세요."**

- **하이브리드 검색**: 키워드 + 의미(벡터) 검색을 결합하여 최적의 결과 제공
- **유사도 점수**: 각 검색 결과의 관련도를 수치로 표시
- **출처 명시**: 어느 문서의 어느 부분인지 정확히 표시
- 텍스트 검색뿐 아니라 "이와 비슷한 내용"을 찾는 의미 기반 검색 가능

### 2.4 사용자 관리

**"역할 기반 접근 제어로 보안을 유지합니다."**

| 기능 | 설명 |
|------|------|
| **역할 분리** | 관리자 / 일반 사용자 권한 분리 |
| **JWT 인증** | 토큰 기반 안전한 인증 체계 |
| **비밀번호 변경** | 사용자 본인이 직접 비밀번호 변경 가능 |
| **계정 관리** | 관리자가 계정 생성/수정/비활성화/삭제 |
| **세션 관리** | 자동 로그아웃, 토큰 갱신 |

### 2.5 관리자 기능

- **사용자 관리**: 계정 생성, 역할 변경, 활성/비활성 토글, 비밀번호 초기화
- **문서 관리**: 전체 문서 열람, 삭제 권한
- **시스템 모니터링**: 헬스체크 API (DB, AI 엔진 상태 확인)

                    │  │      │→│          │  │  :5432   │ │
                    │  └──┬───┘  └──────────┘  └────┬─────┘ │
                    │     │                         │       │
                    │     │     ┌──────────┐        │       │

                    │           │ FastAPI  │                │

### 3.2 기술 스택

| 계층 | 기술 | 설명 |
|------|------|------|
| **프론트엔드** | React 18 + Tailwind CSS | 프리미엄 UI, 모바일 반응형 |
| **백엔드** | FastAPI (Python 3.11) | 비동기 REST API, SSE 스트리밍 |
| **데이터베이스** | PostgreSQL 16 + pgvector | 관계형 DB + 벡터 검색 |
| **AI 엔진** | Ollama | 로컬 LLM 및 임베딩 실행 환경 |
| **LLM 모델** | Qwen2.5 7B | 한국어 성능 우수 (7B 파라미터) |
| **임베딩 모델** | BGE-M3 | 다국어 1024차원 벡터 임베딩 |
| **인증** | JWT (bcrypt) | 토큰 기반 인증, 비밀번호 해싱 |
| **배포** | Docker Compose | 원클릭 배포, 컨테이너 오케스트레이션 |

### 3.3 RAG (검색증강생성) 파이프라인

**RAG**란 AI가 답변할 때 관련 문서를 먼저 검색한 후, 그 내용을 참고하여 답변을 생성하는 기술입니다.

```
[문서 등록 단계]                        [질문 응답 단계]
                                        
 문서 업로드                              질문 입력
     │                                      │
     ▼                                      ▼
 텍스트 추출                             질문 벡터화
 (PDF/Word/Excel)                        (BGE-M3)
     │                                      │
     ▼                                      ▼
 텍스트 청킹                            유사도 검색
 (500자 단위)                           (코사인 유사도)
     │                                      │
     ▼                                      ▼
 벡터 임베딩                            관련 문서 Top-5
 (BGE-M3, 1024차원)                         │
     │                                      ▼
     ▼                                 프롬프트 생성
 DB 저장                               (컨텍스트 + 질문)
 (PostgreSQL + pgvector)                    │
                                            ▼
                                        LLM 답변 생성
                                        (Qwen2.5)
                                            │
                                            ▼
                                        실시간 스트리밍
                                        (SSE)
```

- 드래그 앤 드롭 업로드 영역

## 5. 배포 및 운영

### 5.1 배포 방식

| 방식 | 설명 | 권장 환경 |
|------|------|----------|
| **Docker Compose** | 원클릭 배포, 5개 컨테이너 자동 구성 | 프로덕션 (권장) |
| **로컬 설치** | 개별 서비스 수동 설치 | 개발/테스트 |

### 5.2 Docker Compose 배포 (3단계)

```bash
# 1단계: 환경 설정
cp .env.example .env && vi .env

# 2단계: 시스템 시작
docker-compose up -d

# 3단계: AI 모델 준비
docker exec baikal-ollama ollama pull qwen2.5:7b
docker exec baikal-ollama ollama pull bge-m3
```

### 5.3 폐쇄망 배포

인터넷이 차단된 환경에서도 설치할 수 있습니다:

```
인터넷 환경                    폐쇄망 환경
┌─────────────┐    USB/망    ┌─────────────┐
│ Docker 이미지│───전송───▶  │ 이미지 로드   │
│ AI 모델 파일 │             │ 모델 로드     │
│ 소스 코드    │             │ 시스템 시작   │
└─────────────┘              └─────────────┘
```

제공되는 스크립트:
- `scripts/export-images.sh` — Docker 이미지 저장
- `scripts/export-models.sh` — AI 모델 저장
- `scripts/import-images.sh` — Docker 이미지 로드
- `scripts/import-models.sh` — AI 모델 로드

### 5.4 시스템 요구사항

| 항목 | 최소 사양 | 권장 사양 |
|------|----------|----------|

| 디스크 | 50GB SSD | 100GB SSD |
| GPU | 없어도 가능 | NVIDIA 8GB+ VRAM |

## 6. 보안 특성

### 6.1 데이터 보안

```
✅ 외부 API 호출 제로 — 모든 처리가 사내 서버에서 완료
✅ 데이터 유출 경로 없음 — 인터넷 연결 불필요
✅ 비밀번호 bcrypt 해싱 — 단방향 암호화, 복호화 불가
✅ JWT 토큰 인증 — 서버 서명 기반, 위변조 불가
✅ 역할 기반 접근 제어 — 관리자/사용자 권한 분리
✅ 세션 자동 만료 — 미사용 시 자동 로그아웃
```

### 6.2 인프라 보안

- Docker 컨테이너 격리
- 내부 네트워크(bridge)로 서비스 간 통신
- 외부 노출 포트 최소화 (80번 포트만 필수)
- DB 직접 접근 차단 가능 (포트 매핑 제거)

### 6.3 규정 준수

- **개인정보보호법**: 외부 서버로 데이터 전송 없음
- **정보보안 규정**: 폐쇄망 운영 가능, 인터넷 차단 환경 지원
- **클라우드 보안 규정**: 온프레미스 단독 운영, 클라우드 미사용

## 7. 활용 시나리오

### 시나리오 1: 사내 규정 문의 시스템

> **"인사/총무/법무 규정을 AI에게 물어보세요"**

- 인사규정, 복무규정, 보안규정, 경비규정 등 각종 사내 규정 업로드
- 직원들이 "연차 몇 일이야?", "출장비 정산은 어떻게 해?" 질문
- AI가 해당 규정을 참조하여 즉시 답변 + 출처 표시

### 시나리오 2: 기술 문서 검색

> **"제품 매뉴얼, 기술 사양서를 빠르게 찾으세요"**

- 제품 매뉴얼, 설계 문서, 기술 사양서 업로드
- 엔지니어가 "제품 X의 내열 온도는?", "모듈 Y 인터페이스 규격은?" 질문
- AI가 관련 기술 문서에서 정확한 수치와 규격을 찾아 답변

### 시나리오 3: 프로젝트 지식 관리

> **"지난 프로젝트의 경험과 노하우를 AI로 활용하세요"**

- 프로젝트 보고서, 회의록, 제안서 업로드
- "작년 A 프로젝트에서 리스크 관리는 어떻게 했지?" 질문
- AI가 관련 보고서에서 리스크 관리 방안을 요약하여 답변

### 시나리오 4: 법률 문서 검토 보조

> **"계약서와 법률 문서의 핵심을 AI가 찾아줍니다"**

- 계약서, 법률 자문 문서, 판례 업로드
- "이 계약에서 손해배상 조항은?", "유사 분쟁 사례가 있나?" 질문
- AI가 관련 조항과 유사 사례를 참조하여 답변
