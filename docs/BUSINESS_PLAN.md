## 1. 사업 개요

### 1.1 사업 아이템 요약

| 항목 | 내용 |
|------|------|
| **사업명** | BAIKAL Private AI |
| **제품 유형** | 폐쇄망 설치형 AI 플랫폼 (On-Premise) |
| **핵심 기술** | RAG(검색증강생성) + 로컬 LLM + 벡터 데이터베이스 |
| **목표 고객** | 보안 규정이 있는 공공기관, 금융사, 국방, 의료, 대기업 |
| **사업 형태** | 소프트웨어 라이선스 + 유지보수 서비스 |

### 1.2 미션과 비전

> **미션**: 보안이 필요한 모든 조직이 외부 클라우드에 의존하지 않고 AI의 혜택을 누릴 수 있도록 한다.

> **비전**: 2028년까지 국내 폐쇄망 AI 플랫폼 시장 점유율 1위를 달성한다.

### 1.3 핵심 가치 제안

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   "데이터가 서버 밖으로 나가지 않는 100% 온프레미스 AI"  │
│                                                         │
│   🔒 보안 최우선    데이터 유출 경로 제로               │
│   📄 문서 기반     사내 DB → AI 지식 자산화             │
│   ⚡ 실시간        SSE 스트리밍 답변                    │
│   🏢 온프레미스    폐쇄망·DMZ·사내망 어디든 설치        │
│   💰 비용 절감     오픈소스 기반, 외부 API 과금 없음     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.4 창업 배경 및 문제 인식

**문제**: 기업·기관의 AI 도입 의지는 높으나, 보안상 외부 AI 서비스(ChatGPT, Claude 등)를 사용할 수 없는 환경이 광범위하게 존재한다.

- **공공기관·국방**: 망분리 의무화로 인터넷 연결 불가
- **금용·보험**: 개인정보보호법·금융감독원 규정으로 외부 전송 금지
- **의료기관**: 환자 정보 외부 유출 원천 차단 의무
- **대기업 R&D**: 핵심 기술 문서의 외부 AI 학습 방지

**해결**: 사내 서버에 완전 설치되어 인터넷 없이 동작하는 AI 플랫폼 BAIKAL Private AI를 공급한다.

## 2. 시장 분석

### 2.1 목표 시장 정의

#### TAM (전체 유효 시장)
- **국내 Enterprise AI 플랫폼 시장**: 2026년 약 8,500억 원
- **글로벌 On-Premise LLM 시장**: 2026년 약 12조 원 (연 35% 성장)

#### SAM (서비스 가능 시장)
- **국내 폐쇄망·망분리 환경 기관**: 약 2,800개 (공공기관 1,200 + 금융권 800 + 대기업 800)
- 해당 기관 AI 도입 예산 기준 약 **2,100억 원 규모**

#### SOM (현실적 획득 가능 시장)
- 2026년: 15개 기관 (초기 레퍼런스 확보) → 매출 약 **7.5억 원**
- 2027년: 60개 기관 → 매출 약 **30억 원**
- 2028년: 200개 기관 → 매출 약 **100억 원**

### 2.2 시장 트렌드

| 트렌드 | 영향 |
|--------|------|
| **생성형 AI 도입 급증** | 전 산업 AI 수요 폭발적 증가 |
| **데이터 주권 법제화** | EU AI Act, 국내 개인정보보호법 강화 |
| **망분리 규정 유지** | 공공기관·금융권 외부 AI 서비스 사용 불가 구조 고착 |
| **오픈소스 LLM 고성능화** | Qwen, Llama 등 오픈소스 모델 급성장 → 라이선스 비용 없이 GPT급 성능 |
| **RAG 기술 성숙** | 환각(hallucination) 감소, 기업 문서 기반 AI 신뢰도 상승 |

### 2.3 고객 세그먼트

#### 1순위 — 공공기관 (망분리 의무)
- 중앙행정기관, 지방자치단체, 공기업
- 망분리 義務로 외부 서비스 사용 불가
- 정부 IT 예산 집행 구조 (연간 계획 + 조달청 수의계약)

#### 2순위 — 금융·보험
- 은행, 증권사, 보험사, 카드사
- 금융감독원 가이드라인: 고객 데이터 외부 서버 전송 금지
- IT 투자 여력 풍부, ROI 민감

#### 3순위 — 의료기관·제약
- 상급종합병원, 제약 R&D 센터
- 환자 정보(EMR) 외부 전송 원천 차단
- 연구 문서, 임상 프로토콜 AI 활용 필요

#### 4순위 — 대기업 R&D·제조
- 비밀 설계 문서, 특허 문서 AI 처리 필요
- 핵심 기술 유출 방지 목적 온프레미스 선호

### 2.4 고객 페인포인트

| 고객 유형 | 핵심 페인포인트 | BAIKAL 해결책 |
|-----------|---------------|---------------|
| 공공기관 | ChatGPT 등 사용 불가, 수백 개 규정집 검색 비효율 | 폐쇄망 설치 + RAG 문서 검색 |
| 금융 | 고객 데이터 AI 활용 불가, 챗봇 성능 한계 | 온프레미스 설치 + 금융 문서 기반 답변 |
| 의료 | EMR 데이터 외부 AI 학습 불가, 의학 문서 검색 비효율 | 병원 서버 설치 + 의학 문서 RAG |
| 대기업 | 기술 문서 외부 유출 우려, 내부 전문 지식 검색 어려움 | 사내망 설치 + 기술 DB화 |

#### 주요 기능 모듈

| 모듈 | 기능 설명 |

| **RAG 엔진** | 텍스트 추출 → 청킹 → 임베딩 → 벡터 검색 → LLM 답변 |

| **시스템 모니터링** | 헬스체크 API, DB/AI 엔진 상태 확인 |

### 3.3 서비스 구성 (소프트웨어 + 구축 + 유지보수)

```
Phase 1 — 납품·설치 (2주)
  ├─ 환경 분석 및 아키텍처 설계
  ├─ Docker 이미지 + AI 모델 파일 납품 (오프라인 USB)

  └─ 기본 테스트 및 인수인계

  ├─ 조직 로고·색상 브랜딩 적용
  ├─ 사내 LDAP/AD 인증 연동
  ├─ 부서별 문서 접근 권한 설정
  └─ 초기 문서 데이터 마이그레이션

Phase 3 — 안정화 및 유지보수
  ├─ 월간 정기 점검 (원격/방문)
  ├─ 소프트웨어 업데이트 제공
  ├─ 사용자 교육 (1회 포함)
  └─ 기술 지원 (이메일/전화)
```

### 5.3 경쟁 우위

1. **완전한 폐쇄망 지원**: 인터넷 연결 0, USB 설치 가능 → 경쟁사 미충족 니즈
2. **소스코드 100% 납품**: 기관 자체 수정·운영 가능 → 공공기관 의존성 리스크 없음
3. **낮은 진입 비용**: 오픈소스 LLM 기반으로 라이선스 부담 없음
4. **빠른 구축**: Docker Compose 기반 2주 내 설치 완료
5. **전문 기술 지원**: 국내 팀 → 언어·시간대 장벽 없는 신속 대응

### 7.1 Go-to-Market 전략

#### 1단계: 레퍼런스 확보 (2026 H1)

- 조달청 나라장터 소프트웨어 등록

- 공공기관 IT 구매 담당자 대상 웨비나 운영

#### 3단계: 영업 확장 (2027~)
- 금융·의료 버티컬 전문 파트너사 모집
- 중견기업 IT 유통 채널(VAR) 활용
- 정부 AI 정책 연계 (디지털플랫폼정부, AI허브 등)
