## 1. 제안 배경 및 목적

### 1.1 배경

최근 ChatGPT, Claude 등 생성형 AI의 급속한 발전으로 업무 자동화와 지식 검색의 패러다임이 변화하고 있습니다. 그러나 공공기관, 금융사, 의료기관, 방위산업 등 **보안이 핵심인 환경**에서는 외부 클라우드 AI 서비스 사용이 원칙적으로 불가합니다.

```
현재 상황                          발생하는 문제
────────────────────────────────────────────────────────
• 망분리 의무 환경                  • ChatGPT 등 외부 AI 사용 불가
• 기밀 문서 외부 전송 금지          • 수백 건 문서를 수작업 검색
• 내부 지식 DB 낙후                • 신규 직원 업무 파악 오래 걸림
• AI 활용 요구 급증                 • 관련 부서에 반복 문의 과다
```

### 1.2 제안 목적

BAIKAL Private AI는 **외부 인터넷 연결 없이 사내 서버에서만 동작**하는 AI 플랫폼을 제공하여, 보안 규정을 100% 준수하면서 동시에 생성형 AI의 업무 혁신 효과를 누릴 수 있도록 합니다.

### 1.3 제안 범위

| 범위 | 내용 |
|------|------|
| **소프트웨어 납품** | BAIKAL Private AI Platform v1.1 |
| **시스템 구축** | 서버 설치, 환경 구성, 초기 데이터 마이그레이션 |
| **커스터마이징** | 조직 브랜딩, 인증 연동, 권한 설정 |

## 3. 솔루션 개요

### 3.1 BAIKAL Private AI 소개

**BAIKAL Private AI**는 기업·기관의 사내 문서를 기반으로 AI가 질문에 답변하는 **100% 온프레미스 AI 플랫폼**입니다.

```
          ┌─ 문서 업로드 ─ PDF / Word / Excel
          │
사용자 ──▶│─ 자연어 질문 ─ "이 계약서의 위약금 조항은?"
          │
          └─ AI 답변     ─ "제5조에 따르면 위약금은..."
                                      + 📄 출처 문서 표시

      ※ 모든 데이터 처리가 사내 서버에서만 이루어집니다
```

### 3.2 경쟁 대비 차별점

| 차별점 | 설명 |
|--------|------|
| **완전 폐쇄망 지원** | USB 오프라인 설치, 인터넷 연결 불필요 |
| **소스코드 납품** | 100% 소스코드 제공으로 자체 유지보수 가능 |
| **낮은 도입 비용** | 오픈소스 LLM 기반, 월 API 과금 없음 |
| **빠른 도입** | Docker Compose 기반, 2~4주 내 운영 가능 |
| **한국어 최적화** | Qwen2.5:7b 모델 — 국내 문서 처리 성능 우수 |
| **프리미엄 UI** | 통합 다크 테마, 모바일 반응형, 직관적 인터페이스 |

## 5. 주요 기능 상세

### 5.1 AI 질의응답 (채팅)

**자연어로 사내 문서에 질문하고 즉시 답변을 받습니다.**

| 기능 | 설명 |
|------|------|
| 자연어 질문 | 정해진 형식 없이 일상 언어로 질문 |
| 실시간 스트리밍 | 토큰 단위 실시간 응답 (타이핑 효과) |
| 참고 문서 표시 | AI 답변 근거 문서명 및 내용 일부 표시 |
| 대화 세션 | 대화 이력 저장, 세션별 관리 |
| 다중 세션 | 사용자별 여러 대화 세션 동시 관리 |

**실제 사용 예시**:
```
사용자: 개인정보 처리 방침에서 보유 기간은 어떻게 되나요?

AI:    개인정보처리방침 제4조(개인정보의 보유 및 이용기간)에 따르면,
       수집된 개인정보는 다음 기간 동안 보존됩니다:
       
       · 회원 정보: 회원 탈퇴 후 30일
       · 계약/청약 철회 기록: 5년 (전자상거래법)
       · 대금 결제 기록: 5년 (전자상거래법)
       · 소비자 불만 처리 기록: 3년 (전자상거래법)
       
       📄 출처: 개인정보처리방침_2025.pdf
```

## 6. 구축 방법론 및 일정

### 6.1 구축 단계

```
Phase 1 — 착수 / 환경 분석 (1주)
  ├─ 킥오프 미팅
  ├─ 서버 환경 분석 (OS, 사양, 네트워크 구성)
  ├─ 기존 문서 현황 파악
  └─ 납품 및 설치 계획 확정

Phase 2 — 설치 및 구성 (1주)
  ├─ Docker 이미지 패키지 반입 (USB)
  ├─ AI 모델 파일 반입 및 로드
  ├─ 서비스 컨테이너 기동 및 테스트
  └─ 기본 설정 (관리자 계정, 환경변수)

Phase 3 — 커스터마이징 (1~2주)
  ├─ 조직 로고 및 브랜딩 적용
  ├─ 사용자 계정 일괄 생성
  ├─ 인증 연동 (LDAP/AD, 요청 시)
  └─ 초기 문서 데이터 일괄 업로드

Phase 4 — 테스트 및 교육 (1주)
  ├─ 기능 검수 (체크리스트 기반)
  ├─ 성능 테스트 (동시 사용자 부하 테스트)
  ├─ 관리자 교육 (4시간)
  ├─ 사용자 교육 (2시간)
  └─ 인수인계 및 운영 매뉴얼 전달
```

### 6.2 표준 구축 일정 (4주)

| 주차 | 작업 항목 | 산출물 |
|------|----------|-------|

## 13. FAQ

**Q1. 기존 전사 포털이나 ERP와 연동이 가능한가요?**  
A. 현재 버전은 독립 동작 방식이며, API 형태의 연동은 커스터마이징으로 가능합니다. SSO 연동(LDAP/AD)은 v1.2에서 기본 지원 예정입니다.

**Q2. 문서가 수천 건 이상인 경우 성능이 괜찮나요?**  
A. pgvector는 수십만 건의 벡터도 밀리초 단위로 검색합니다. 문서 10,000건(청크 수 ~50,000개) 수준에서 검색 응답 200ms 이내를 목표로 최적화합니다. 대용량 시 인덱스 튜닝을 지원합니다.

**Q3. AI 모델을 교체하거나 추가할 수 있나요?**  
A. Ollama를 통해 qwen2.5 외에도 Llama3, Mistral 등 다양한 오픈소스 모델로 교체하거나 추가 설치가 가능합니다. 모델 선택 UI는 v1.2에서 제공 예정입니다.

**Q4. 한글(HWP) 문서는 지원되나요?**  
A. 현재 버전에서는 PDF/DOCX/XLSX를 지원합니다. HWP 지원은 v1.2(2026 Q3)에서 제공 예정이며, 별도 커스터마이징으로 빠른 제공도 가능합니다.

**Q5. 인터넷이 완전히 차단된 환경에서도 설치 가능한가요?**  
A. 네, 전혀 문제없습니다. 제공되는 USB 패키지(Docker 이미지 + AI 모델)를 이용해 인터넷 연결 없이 완전 설치가 가능합니다. 이것이 BAIKAL의 핵심 차별점입니다.

**Q6. GPU 서버가 없어도 사용할 수 있나요?**  
A. 가능합니다. CPU 전용 모드로 동작하며, 이 경우 QWen2.5:7b 모델 기준 답변 생성 속도가 다소 느릴 수 있습니다(약 5~15토큰/초). GPU(NVIDIA CUDA) 환경에서는 50토큰/초 이상 가능합니다.

**Q7. 소스코드를 납품받으면 자체 수정이 가능한가요?**  
A. 네, 100% 소스코드를 납품합니다. Python(FastAPI), JavaScript(React), SQL 지식을 보유한 내부 개발자가 있다면 자체적으로 기능을 추가·수정할 수 있습니다. 단, 수정된 버전에 대한 당사 기술 지원은 계약 범위와 별도로 협의합니다.

**Q8. 동시 접속자가 많으면 어떻게 되나요?**  
A. Professional 버전 기준 동시 접속 30명이 원활하게 사용 가능합니다. 그 이상의 동시 사용자 환경은 Enterprise 버전(HA 구성)을 권장합니다.
